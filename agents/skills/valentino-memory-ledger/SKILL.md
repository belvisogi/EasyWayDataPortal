# Valentino Memory Ledger

Use this skill to keep continuity across sessions.

## Memory File
- `docs/ops/VALENTINO_EXECUTION_MEMORY.md`

## What Must Be Recorded
- Decision taken (and why)
- Active constraints and non-negotiables
- API contract changes
- Open risks/blockers
- Next 1-3 actions

## Workflow
1. Read latest memory file section.
2. Append a new dated entry (do not rewrite history).
3. Keep entries short and factual.
4. Link affected files.

## Entry Format
1. Date
2. Context
3. Decision
4. Risks
5. Next actions
